🔐 HOW TO ADD ADMIN USERS (Firebase Console)

Since admin signup is now disabled in the app for security reasons,
you need to manually add admin users through Firebase Console.

═══════════════════════════════════════════════════════════

METHOD 1: Create Admin Account via Firebase Console
═══════════════════════════════════════════════════════════

STEP 1: Create the User Account
────────────────────────────────────────────────────────────
1. Go to Firebase Console:
   https://console.firebase.google.com/project/al-ameen-stable/authentication/users

2. Click "Add user" button

3. Enter:
   - Email: admin@alameenstable.com (or your admin email)
   - Password: (create a strong password)

4. Click "Add user"

5. COPY the User UID that appears (looks like: abc123def456...)


STEP 2: Set User Role to Admin
────────────────────────────────────────────────────────────
1. Go to Firestore Database:
   https://console.firebase.google.com/project/al-ameen-stable/firestore

2. Click on "users" collection (or create it if it doesn't exist)

3. Click "Add document"

4. Set Document ID to the User UID you copied in Step 1

5. Add these fields:
   Field Name    | Type   | Value
   ─────────────────────────────────────
   email         | string | admin@alameenstable.com
   name          | string | Admin Name
   role          | string | admin
   createdAt     | string | 2025-10-10

6. Click "Save"


STEP 3: Test Admin Login
────────────────────────────────────────────────────────────
1. Open your app
2. Click "Sign In"
3. Use the admin email and password you created
4. You should see the Admin Tabs with all management features!

═══════════════════════════════════════════════════════════

METHOD 2: Upgrade Existing User to Admin
═══════════════════════════════════════════════════════════

If someone already has a client account and you want to make them admin:

1. Go to Firestore:
   https://console.firebase.google.com/project/al-ameen-stable/firestore

2. Click on "users" collection

3. Find their user document (search by email)

4. Click on the document

5. Find the "role" field

6. Change the value from "client" to "admin"

7. Click "Update"

8. Next time they log in, they'll have admin access!

═══════════════════════════════════════════════════════════

🔒 SECURITY BENEFITS:

✅ Only authorized people can become admins
✅ No random users can self-promote to admin
✅ You have full control over who manages the stable
✅ Admins must be explicitly granted access by you

For your stable, consider creating these admin accounts:

- manager@alameenstable.com (stable manager)
- owner@alameenstable.com (owner access)

═══════════════════════════════════════════════════════════

⚠️ IMPORTANT NOTES:

1. Keep admin passwords secure and strong
2. Don't share admin credentials
3. Regular clients can only sign up as "client" role
4. You can have multiple admins
5. Admins can be demoted back to clients by changing their role
